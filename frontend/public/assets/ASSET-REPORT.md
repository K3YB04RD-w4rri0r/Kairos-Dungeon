# MistralWorld Asset Report

Generated by the asset processing pipeline (`tools/assets/`).

## 1. Purpose & Context

MistralWorld is a roguelike dungeon crawler requiring pixel art sprites, tiles, UI elements, and particle effects. This asset pipeline downloads CC0 open-source art packs and transforms them into game-ready sprite sheets matching the `docs/schemas/assets/sprite-manifest.json` specification.

**Pre-baked Variants:** All 126 creature sprite sheets (6 body types x 7 materials x 3 sizes) are generated as individual PNG files. Materials apply a multiplicative color tint preserving the original DCSS shading. Sizes use nearest-neighbor upscaling from 32x32 source tiles.

## 4. Processing Details

### Frame Mapping
Source tiles selected from DCSS by keyword matching (skeleton->biped, hound->quadruped, snake->serpent, spider->insect, jelly->blob, bat/dragon->winged). See `tools/assets/frame_mapping.json`.

### Animation Generation from Static Poses
- **Idle:** 1px vertical bob (ping-pong: [0, -1, 0, 1])
- **Walk:** Horizontal shifts [0, 1, 2, 1, 0, -1] with vertical bob
- **Attack:** Pull-back/strike sequence [-2, 0, 3, 0] horizontal
- **Death:** Progressive rotation [0, 15, 30, 45 degrees] + opacity fade [255, 200, 130, 60]
- **Summon_in:** Opacity ramp [0, 50, 100, 160, 210, 255]

### Material Colorization
Creatures use original DCSS colors for the flesh material (no processing). Non-flesh materials apply luminance-preserving colorization:
1. Extract per-pixel BT.601 luminance from the original RGB
2. Normalize luminance to [0.05, 0.95] using the frame's min/max (stretches contrast on dark DCSS tiles)
3. Map through a two-ramp gradient: black → tint color (at lum 0.5) → white
This preserves shading structure while producing clearly visible material colors regardless of source brightness.

### Upscaling
All upscaling uses nearest-neighbor (Pillow `Image.NEAREST`) to preserve pixel art crispness.
